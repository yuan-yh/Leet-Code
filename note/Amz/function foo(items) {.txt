Design & implement a LoggerRateLimit system that receives messages with timestamps and returns whether the message should be printed.
The system should follow these rules:

- Each unique message can only be printed at most every 10 seconds
- If a message arrives and it's been 10 seconds since it was last printed, it should be printed
- Given a message and timestamp, return true if the message should be printed, false otherwise
- You need to support querying what messages were printed in a given time range

Examples:
logger.shouldPrintMessage(1, "foo"); // return true, print "foo"
logger.shouldPrintMessage(2, "bar"); // return true, print "bar"
logger.shouldPrintMessage(3, "foo"); // return false, don't print "foo"
logger.shouldPrintMessage(11, "foo"); // return true, print "foo"
logger.getMessagesPrintedInRange(1, 5); // return ["foo", "bar"]
