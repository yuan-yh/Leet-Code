This article discusses the design ideas of Tool and MCP, and how they help Agents achieve specific goals in different scenarios. Key points include:​

1.

Agent Tool Requirements : Different Agents have different tool requirements. For example, the core requirement of the Dialogue Agent is intelligent Q&A interaction, and it needs the query\_internal\_docs tool. The core requirements of the Operation and Maintenance Agent are automatic alarm response, log/monitoring linkage troubleshooting, and root cause analysis, and it needs tools such as query\_prometheus\_alerts, query\_log, get\_current\_time, and query\_internal\_docs. ​

2.

query\_prometheus\_alerts : Its function is to obtain Prometheus alarm information in real - time. It achieves this by API docking with Prometheus and extracting key fields from the JSON response. It has strong real - time performance.​

3.

query\_internal\_docs : It is used to retrieve internal documents. It stores documents in a vector database and is automatically triggered by the large - model when background knowledge is needed. It can avoid the "hallucination" of the large - model.​

4.

get\_current\_time : It solves the "time amnesia" problem of the large - model. It has a minimalist interface and is automatically pre - called by the Agent when time parameters are needed. It is lightweight and efficient.​

5.

query\_log : It enables natural - language - driven log retrieval. By integrating with Tencent Cloud CLS MCP, it allows non - technical personnel to query logs in daily language and dynamically adapts to different log sources.​

6.

Typical Scenario Example : Taking the service offline alarm as an example, the Agent can complete tasks such as data collection, time calibration, log backtracking, knowledge supplementation, and decision - making output through the cooperation of various tools and MCP.​

7.

Overall Value : Through standardized Tool interfaces and flexible MCP integration, the large - model can be upgraded from passive dialogue to active diagnosis, realizing the practical value of AI in the operation and maintenance scenario. ​

MCP 集成​

query\_log ：自然语言驱动的日志检索​

功能定位 ： 让运维/业务人员用日常语言查询日志，降低技术门槛​

实现细节：​

•MCP 对接 ：集成腾讯云CLS MCP，由MCP自动将自然语言转为查询语句，统一返回结构化日志结果​

•调用逻辑 ：Agent 将用户问题（如 查询服务下线前 5 分钟的错误日志 ）直接转发给 CLS MCP，MCP 自动生成日志查询语句并返回结果，Agent 再将结果整理后提交给大模型。​

核心优势​

•低门槛 ：让业务人员用日常语言查日志（如 服务下线前5分钟的错误日志 ），无需学Lucene/SQL语法​

•动态适配 ：MCP 自动处理不同日志源的语法差异，统一输出格式。​

​

典型场景举例：服务下线告警处理流程​

以 服务下线 告警为例，展示 Agent 如何协同 Tool 与 MCP 完成任务：​​

数据采集 ：调用query\_prometheus\_alerts获取告警详情（名称、触发时间）；​

时间校准 ：通过get\_current\_time计算告警持续时长（如 已持续11分钟 ）；​

日志回溯 ：调用query\_log传入 广告微服务 下线前5分钟 错误日志 ，CLS MCP返回关键日志片段；​

知识补充 ：大模型触发query\_internal\_docs，检索 服务下线 处理步骤 文档，获取标准化操作指南；​

决策输出 ：整合上述数据，生成含日志摘要、处理步骤、历史案例的故障报告。​

​

总结​

通过标准化Tool接口（实时告警数据、知识检索、时间感知）与灵活MCP集成（自然语言查询日志），让大模型从 被动对话 升级为 主动诊断 ，高效衔接监控数据、内部知识与业务系统，真正实现AI在运维场景的落地价值​
